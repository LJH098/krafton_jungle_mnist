## 1. 프로젝트 요약

MNIST 10-class 손글씨 숫자 분류기를 **NumPy만으로 직접 구현한 신경망**으로 학습했습니다. PyTorch, TensorFlow 같은 딥러닝 프레임워크 없이 Affine, BatchNorm, ReLU, Dropout, Softmax, Cross Entropy, Adam을 직접 구현하고 조합하여 97% 목표 정확도를 넘기는 것이 핵심 목표였습니다.

| 핵심 항목 | 결과 |
| --- | --- |
| 과제 목표 | MNIST 테스트 정확도 97% 이상 |
| 최종 테스트 정확도 | **98.56%** |
| 최종 모델 파라미터 수 | **537,354개** |
| 학습 시간 | 317초 |
| 주요 시도 | 기본 MLP 구현, 하이퍼파라미터 탐색, 은닉층 경량화 실험 |

> 발표 멘트: 이 프로젝트는 딥러닝 프레임워크 없이 NumPy만으로 MNIST 분류 신경망을 끝까지 구현하고, 목표였던 97%를 넘어 98.56% 테스트 정확도를 달성한 실험입니다.

---

## 2. 모델 구조

최종 모델은 28×28 이미지를 784차원 벡터로 펼친 뒤, 두 개의 은닉층과 하나의 출력층을 통과하는 완전연결 신경망입니다. 각 은닉층에는 Affine, BatchNorm, ReLU, Dropout을 순서대로 적용했습니다.

| 구분 | 내용 |
| --- | --- |
| **입력** | 784차원 입력, 28×28 픽셀을 0~1 범위로 정규화 |
| **은닉층 1** | Affine(784 → 256) → BatchNorm → ReLU → Dropout |
| **은닉층 2** | Affine(256 → 128) → BatchNorm → ReLU → Dropout |
| **출력층** | Affine(128 → 10) → Softmax |
| **손실 함수** | Cross Entropy Loss |
| **가중치 초기화** | He initialization, bias는 0으로 초기화 |

> 발표 멘트: 모델은 단순한 완전연결 구조이지만, BatchNorm과 Dropout을 함께 넣어서 학습 안정성과 일반화 성능을 확보했습니다.

### 이미지 해석

이 이미지는 최종 제출 모델의 학습 손실 곡선입니다. 첫 epoch 이후 손실이 급격히 감소하고, 이후 20 epoch까지 완만하게 줄어드는 형태를 보입니다. 그래프가 중간에 발산하지 않고 안정적으로 내려가기 때문에 학습률과 Adam optimizer 설정이 과도하지 않았다고 해석할 수 있습니다. 후반부에서는 손실 감소 폭이 작아지며 수렴 구간에 진입했으므로, 20 epoch 설정은 최종 성능 확인에 충분했습니다.

> 발표 멘트: 손실은 초반에 빠르게 떨어지고 후반에는 완만하게 수렴했습니다. 최종 테스트 정확도는 98.56%로 목표였던 97%를 안정적으로 넘었습니다.

## 6. 개선

### 6.1 최적 하이퍼파라미터 탐색

기본 모델이 이미 98%대 정확도에 도달했기 때문에, 추가 성능 향상을 위해 단일 설정에 의존하지 않고 learning rate, Dropout 비율, batch size, 은닉층 구조를 함께 탐색했습니다. 탐색은 validation accuracy를 기준으로 후보를 줄이고, 상위 5개 조합을 20 epoch 동안 재학습한 뒤 test accuracy로 최종 비교했습니다.

| 탐색 항목 | 후보 |
| --- | --- |
| Learning rate | 0.003, 0.001, 0.0007, 0.0005, 0.0003 |
| Dropout 비율 | 0.2, 0.3, 0.4, 0.5 |
| Batch size | 64, 128, 256 |
| 은닉층 구조 | 1층~4층, 다양한 뉴런 수 조합 |
| 후보 선정 기준 | Validation Accuracy |

![하이퍼파라미터 탐색 요약](<assets/image (5).png>)

#### 이미지 해석

이 이미지는 하이퍼파라미터 탐색 단계의 전체 요약입니다.

- 왼쪽 위의 막대그래프는 validation accuracy 기준 상위 10개 조합을 보여줍니다. 상위권 조합의 validation accuracy가 95.66%~95.88%로 매우 촘촘하게 모여 있어, 단일 파라미터 하나보다 조합 전체의 균형이 중요하다는 점을 보여줍니다.
- 오른쪽 위의 선 그래프는 상위 5개 후보의 짧은 탐색 학습 구간에서 training loss가 얼마나 빠르게 감소하는지 비교합니다. 모든 후보가 초반 손실을 빠르게 낮추지만, 감소 속도와 최종 손실은 구조별로 차이가 있습니다.
- 왼쪽 아래의 depth comparison은 은닉층 깊이에 따른 평균 성능과 최고 성능을 비교합니다. 평균 기준으로는 3층 구조가 가장 높고, 최고 성능도 3층에서 가장 높게 나타났습니다.
- 오른쪽 아래의 표는 상위 후보의 구조, learning rate, dropout, batch size를 한 번에 정리합니다. 최고 후보는 `1024 -> 512 -> 256`, learning rate `0.0007`, dropout `0.3`, batch size `64` 조합입니다.

> 발표 멘트: 짧은 탐색에서는 3층 구조가 가장 좋은 validation 성능을 보였고, 특히 1024, 512, 256으로 줄어드는 구조가 가장 유력한 후보로 선정됐습니다.

#### 이미지 해석

이 이미지는 탐색에서 뽑힌 상위 5개 조합을 20 epoch 동안 재학습한 결과입니다. 왼쪽 그래프는 각 후보의 training loss 곡선이고, 오른쪽 그래프는 최종 test accuracy를 비교한 막대그래프입니다.

Top 1인 `1024 -> 512 -> 256` 구조는 test accuracy **98.53%**로 가장 높은 일반화 성능을 보였습니다. 반면 Top 2는 final train loss가 **0.0162**로 더 낮았지만 test accuracy는 98.45%였습니다. 즉, 학습 데이터에서 손실을 더 낮게 만드는 것이 항상 test accuracy 향상으로 이어지지는 않았습니다. 이 결과는 모델 선택에서 train loss만 보는 것이 아니라 validation/test 성능을 함께 봐야 한다는 점을 보여줍니다.

> 발표 멘트: Top 2는 학습 손실이 더 낮았지만 테스트 정확도는 Top 1이 더 높았습니다. 그래서 최종 판단은 train loss가 아니라 일반화 성능을 기준으로 했습니다.

#### 하이퍼파라미터 탐색 결론

최적화 실험에서 가장 높은 test accuracy를 보인 조합은 다음과 같습니다.

| 항목 | 선택값 |
| --- | --- |
| 은닉층 구조 | 1024 → 512 → 256 |
| Learning rate | 0.0007 |
| Dropout 비율 | 0.3 |
| Batch size | 64 |

이 조합은 성능이 높았지만, 현재 최종 제출 모델의 98.56%와 비교하면 정확도 차이는 매우 작습니다. 따라서 이 실험의 핵심 의미는 단순히 모델을 키우는 것보다 은닉층 깊이, learning rate, Dropout, batch size의 조합이 성능에 영향을 준다는 점을 확인한 것입니다. 또한 MNIST에서 완전연결 신경망만으로 98%대 후반 이후의 성능을 크게 끌어올리기는 어렵기 때문에, 99% 이상을 목표로 한다면 CNN처럼 이미지의 공간 구조를 활용하는 모델이 더 적합합니다.

> 발표 멘트: 하이퍼파라미터 탐색으로 98.5%대 성능을 재현했지만, 완전연결 신경망만으로는 추가 향상 폭이 제한적이었습니다. 더 높은 목표에는 CNN 도입이 필요하다고 판단했습니다.

### 6.2 은닉층 1개 경량화 시도

추가로 은닉층을 1개만 사용하는 경량 모델을 실험했습니다. 목적은 정확도를 무조건 높이는 것이 아니라, 파라미터 수와 연산량을 줄이면서 목표 정확도 97%를 넘길 수 있는 최소 규모를 찾는 것이었습니다.

![은닉층 크기별 학습 손실](<assets/image (8).png>)

#### 이미지 해석

이 이미지는 은닉층 1개 구조에서 hidden size를 8부터 256까지 바꿔가며 training loss를 비교한 그래프입니다. hidden size가 작을수록 손실이 높게 남고, hidden size가 커질수록 손실이 더 낮아집니다. 특히 8, 16은 손실이 충분히 낮아지지 않아 표현력이 부족하고, 32 이후부터 손실이 크게 개선됩니다. 128, 256은 가장 낮은 손실을 보이지만, 그만큼 모델 크기와 연산량도 증가합니다.

> 발표 멘트: 은닉층이 하나뿐이면 hidden size가 모델의 표현력을 거의 결정합니다. 32 이하에서는 부족하고, 64 이상부터 안정적인 성능이 나오기 시작했습니다.

![파라미터 수 대비 정확도](<assets/image (6).png>)

#### 이미지 해석

이 이미지는 은닉층 1개 모델에서 파라미터 수와 test accuracy의 관계를 보여줍니다. 빨간 점선은 97% 목표 정확도입니다. hidden size 8과 16은 목표에 미치지 못했고, hidden size 32는 97%에 근접하지만 아직 부족합니다. hidden size 40부터 목표선을 넘기 시작하며, 64 이후에는 정확도 향상 폭이 점점 작아집니다. 즉, 파라미터 수를 크게 늘릴수록 정확도는 오르지만, 일정 지점 이후에는 추가 파라미터 대비 성능 향상이 제한됩니다.

> 발표 멘트: 파라미터 수 기준으로 보면 hidden size 40 근처가 목표 정확도 97%를 넘기는 최소 구간이고, 128 이상부터는 성능 향상 대비 모델 크기 증가가 커집니다.

![추론 FLOPs 대비 정확도](<assets/image (7).png>)

#### 이미지 해석

이 이미지는 추론 1회당 FLOPs와 test accuracy의 관계를 보여줍니다. 파라미터 수 그래프와 거의 같은 형태를 보이며, hidden size가 커질수록 연산량과 정확도가 함께 증가합니다. 하지만 목표 정확도 97%를 넘긴 이후에는 연산량 증가에 비해 정확도 상승 폭이 작아집니다. 따라서 실제 배포나 제한된 환경을 고려한다면 가장 높은 정확도만 선택하기보다, 목표 정확도를 넘기는 선에서 연산량이 작은 모델을 고르는 것이 합리적입니다.

> 발표 멘트: 정확도만 보면 큰 모델이 좋지만, 연산량까지 보면 97% 목표를 넘기는 가장 작은 모델을 선택하는 전략도 충분히 의미가 있습니다.

#### 경량화 실험 결론

은닉층 1개 모델은 최종 2층 모델보다 단순하지만, hidden size를 충분히 키우면 97% 목표 정확도를 넘길 수 있었습니다. 이 실험을 통해 모델 성능은 구조의 깊이뿐 아니라 hidden size, 파라미터 수, 추론 연산량과 함께 판단해야 한다는 점을 확인했습니다.

> 발표 멘트: 최종 제출은 2층 모델을 사용했지만, 경량화 실험을 통해 목표 정확도를 만족하는 더 작은 모델의 가능성도 확인했습니다.

## 7. 종합 결론

이번 프로젝트에서는 NumPy만으로 신경망의 핵심 구성 요소를 직접 구현하고, MNIST 분류에서 **98.56%** 테스트 정확도를 달성했습니다. BatchNorm과 Dropout을 포함한 2층 완전연결 신경망은 안정적으로 수렴했고, 목표 정확도 97%를 충분히 넘었습니다.

하이퍼파라미터 탐색에서는 3층 구조와 작은 batch size, 적절한 Dropout 조합이 높은 성능을 보였지만, 완전연결 신경망만으로는 98%대 후반 이후 성능 향상 폭이 제한적이었습니다. 경량화 실험에서는 hidden size가 커질수록 정확도는 올라가지만, 파라미터 수와 FLOPs도 함께 증가한다는 trade-off를 확인했습니다.

| 관점 | 결론 |
| --- | --- |
| 최종 성능 | 98.56%로 목표 97% 초과 달성 |
| 학습 안정성 | 손실 곡선이 발산 없이 수렴 |
| 하이퍼파라미터 탐색 | 3층 `1024 → 512 → 256`, lr 0.0007, dropout 0.3, batch 64가 강한 후보 |
| 경량화 | hidden size 40 이상부터 97% 목표에 도달 가능 |
| 다음 개선 방향 | 99% 이상을 목표로 한다면 CNN 도입 필요 |

> 발표 멘트: 최종적으로 목표 정확도는 달성했고, 추가 실험을 통해 성능을 높이는 방향과 모델을 줄이는 방향을 모두 확인했습니다. 다음 단계는 완전연결 신경망의 한계를 넘어 CNN을 적용하는 것입니다.
